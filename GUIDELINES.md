# Guidelines Techniques

## 🌿 Branching Strategy
Nous suivons une stratégie basée sur les tickets :
- **Branches de fonctionnalités :** `feat/TICKET-ID` (ex: `feat/TICKET-01`)
- **Branches de correction :** `fix/TICKET-ID`
- **Branche principale :** `main` (stable)

## 💬 Convention de Commits (Semantic)
Le format des messages de commit et des Pull Requests doit être :
`type(Scope): TICKET-ID Description`

**Exemples :**
- `feat(Listing): TICKET-01 Ajout d'un tableau pour lister les tâches`
- `fix(API): TICKET-04 Correction du format de date`
- `docs(README): Mise à jour des instructions d'installation`

## 📏 Standards de Code
- **TypeScript** pour le Front.
- **ESLint** configuré.
