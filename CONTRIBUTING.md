# Guide de Contribution

## 🟦 Cycle de vie d'un ticket
Nous utilisons GitHub Projects pour suivre l'avancement. Voici nos étapes :
1. **Backlog** : Idées et besoins bruts.
2. **Ready for Dev** : Ticket répondant à la **DoR**.
3. **In Progress** : Travail en cours (une seule personne assignée).
4. **In Review** : Code terminé, en attente de relecture (Pull Request).
5. **Done** : Validé et fusionné sur `main`.

## 📋 Definition of Ready (DoR)
Un ticket ne peut entrer en Sprint que si :
- Titre clair et actionnable.
- Description (Contexte + Besoin + Résultat).
- Type identifié (Front / Back / Fullstack).
- Estimation en points (1, 2, 3, 5).
- Priorité définie.

## ✅ Definition of Done (DoD)
Un ticket est considéré comme terminé si :
- Code implémenté et fonctionnel.
- Aucun bug bloquant connu.
- Code Review effectuée et validée.
- Branche fusionnée dans `main`.
